# 设计系统与无障碍

**简体中文** · [English](en/design-system-and-accessibility.md)

官网保留高端黑白视觉，CMS 使用高密度深色运营界面。两端通过令牌、状态语言和无障碍规则保持一致，而不是复制同一套组件。

![颜色、间距、圆角与状态令牌](images/design-tokens.svg)

## 视觉原则

- 以黑、石墨、白为主，绿色作为可操作与成功提示，金色只做有限强调。
- 正文、辅助文字、边框和状态色必须满足对应背景的可读对比度。
- 间距按 4/8/12/16/24 的节奏组合；控件圆角约 8px，卡片约 14px。
- 动画使用 160–260ms 的短过渡；信息不能只靠动画出现。

## 状态组件

所有数据页面必须提供：

- loading：明确正在发生什么，避免整页跳动。
- empty：说明为何为空并提供下一步。
- error：使用者能理解的错误与可重试动作。
- success：以服务端成功为准，不提前乐观宣告。
- disabled/permission denied：说明权限或前置条件，不只降低透明度。

## 键盘与焦点

- 所有链接、按钮、表单和标签页可通过键盘到达。
- `:focus-visible` 必须清晰，不能用 `outline: none` 无替代地移除。
- 弹窗打开后聚焦标题或首个有效控件，焦点限制在弹窗内，关闭后返回触发按钮。
- Escape 关闭非破坏性弹窗；破坏性确认不能被意外键盘操作触发。
- 跳转链接、页面标题层级和 landmark 支持屏幕阅读器导航。

## 内容与媒体

- 产品图 alt 描述产品和视角，不堆叠关键词；装饰图使用空 alt。
- 图片声明尺寸或宽高比，防止 CLS；首屏主图避免无意义延迟加载。
- 真实产品照片必须有授权和来源；生成图只能用于明确标注的概念或占位场景。
- 视频提供标题、控制和必要文字替代。

## reduced motion 与响应式

`prefers-reduced-motion: reduce` 下关闭非必要位移、缩放和视差，保留即时状态反馈。验收视口至少覆盖 1440×900、834×1112 与 393×852，并验证 200% 缩放、长文本和触控目标。

## 文档截图规范

- 截图来自重新 seed 的本地三端，只包含虚构数据。
- 桌面使用 1440×900，平板 834×1112，手机 393×852；同类截图保持一致。
- 文件扩展名必须与真实编码一致，UI 截图优先 PNG。
- 每张截图提供准确 alt 和一句业务说明；界面明显变化时同 PR 更新。
