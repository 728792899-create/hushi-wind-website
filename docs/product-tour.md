# 产品与用户旅程

**简体中文** · [English](en/product-tour.md)

这份文档用业务语言说明官网如何把“看一看”变成“可处理的咨询线索”。所有截图均来自本地虚构 Demo 数据。

![从浏览到 CMS 处理的业务路径](images/customer-journey.svg)

## 1. 首页：先建立信任和方向感

![胡氏管乐官网首页](screenshots/01-homepage.jpg)

首页不试图一次展示全部信息，而是完成三件事：

1. 用品牌文案和主视觉建立品类、定位和可靠感。
2. 用 Featured Series 提供有意图的产品入口。
3. 用服务、品牌内容和新闻支撑不同成熟度的访客。

主要 CTA 不直接弹出表单，而是先把访客带到可理解、可比较的产品信息中。

## 2. 产品目录：快速缩小选择范围

![产品搜索与筛选](screenshots/02-products.jpg)

产品目录包含：

- 类目快速筛选。
- 产品名、参数和场景关键词搜索。
- 价格区间、库存状态和排序。
- 结果数、当前筛选语义和清空操作。
- 最多 3 件产品的底部对比栏。

键盘用户可直接访问搜索框、筛选控件、产品卡和对比按钮。手机端把低频筛选收进 drawer，保留搜索和当前结果反馈。

## 3. 选型对比：把差异放在一个可扫读表面

![产品选型对比弹窗](screenshots/07-product-compare.png)

对比弹窗并排展示主图、参考价、库存、系列、类别、关键参数、场景和售后说明。它不代替详情页，而是帮助用户判断下一步应该深入查看哪一款。

可访问性合同：

- 打开后焦点进入关闭按钮。
- Tab/Shift+Tab 不逃出 dialog。
- Escape 关闭。
- 关闭后焦点返回“开始对比”按钮。
- 小屏幕在弹窗内滚动，不会让页面背景误滚。

## 4. 详情与 CTA：在用户决策时提供下一步

| 桌面详情 | 手机详情 |
| --- | --- |
| ![桌面端产品详情和咨询入口](screenshots/11-product-detail-desktop.png) | ![手机端产品详情和固定咨询入口](screenshots/12-product-detail-mobile.png) |

产品详情页按以下层级组织：

1. 产品名、系列、价格/报价模式和库存。
2. 主图、图库和产品故事。
3. 核心特性、结构化参数和适用场景。
4. 资料下载、售后和与该类别关联的 FAQ。
5. 预约试奏、咨询报价和同类产品。

SEO 使用 Product 与 BreadcrumbList JSON-LD，价格、库存和图片信息来自同一产品模型。

## 5. 咨询表单：只在必要时收集必要数据

![产品咨询报价表单](screenshots/08-inquiry-form.png)

表单会自动带入当前产品，让用户补充姓名、联系方式、城市、预算、时间和可选需求。

提交逻辑的重要边界：

- 前端与 API 都进行字段校验。
- 公开表单使用独立限流 bucket，不与普通内容读取共用。
- honeypot 和最短填写时间用于拦截简单机器人。
- `inquiry_submit` 只在 API 创建成功后记录。
- 分析层不收集姓名、联系方式、城市和留言。

## 6. CMS 闭环：咨询必须变成可跟进工作

线索保存后，具备 `crm:read` 权限的角色可在“客户与工单”中查看；具备 `crm:write` 的角色才能标记、更新或删除。导出需要独立 `exports:create` 权限，并保留导出记录。

| CMS 运营工作台 | 咨询队列 |
| --- | --- |
| ![CMS 运营工作台](screenshots/03-admin-dashboard.jpg) | ![CMS 客户与工单中的咨询队列](screenshots/15-cms-crm-inquiries.png) |

运营工作台把线索、紧急工单、内容缺口、资源风险、最近审计和转化漏斗放在同一个决策面。

## 7. 手机与平板路径

| 平板产品列表 | 手机筛选 | 手机导航 |
| --- | --- | --- |
| ![平板产品列表](screenshots/04-tablet-products.jpg) | ![手机筛选](screenshots/05-mobile-products.jpg) | ![手机导航](screenshots/09-mobile-navigation.png) |

响应式验收不只看卡片是否换行，还包括：

- 安全区和底部对比栏不重叠。
- 打开菜单、筛选 drawer 或 modal 后背景不滚动。
- 可操作目标保留足够触控高度。
- 393×852 与 834×1112 视口下没有水平溢出。
- 搜索、筛选、对比和咨询仍可完整到达。

## 8. 验收路径

手工验收：

1. 从首页进入产品目录。
2. 搜索一个型号并清空筛选。
3. 加入两件产品并打开对比。
4. 通过键盘关闭对比并确认焦点返回。
5. 进入产品详情，打开咨询表单。
6. 用明确标记为测试的虚构数据提交。
7. 登录 CMS，查找该线索并标记已读。
8. 重新执行 `npm run seed:demo` 清理演示记录。

自动化等价路径由 `npm run test:e2e` 覆盖。
