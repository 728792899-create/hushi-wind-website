# 面试讲述要点

## 一句话

这是一个小型商业官网交付样板，覆盖前台展示、后台 CMS、API、数据库、本地演示和生产配置校验。

## 核心设计

- 三端分离：Nuxt 前台负责展示，Vue 后台负责运营管理，Express API 负责数据和权限。
- 内容模型：产品、文章、FAQ、页面内容、咨询线索等按业务表拆分。
- 本地演示：通过脱敏 seed 快速生成可登录、可浏览、可编辑的数据。
- 安全意识：生产环境拒绝默认密码、弱 token 和 localhost 构建配置。

## 可以展开的问题

- 为什么官网项目也要保留后台 CMS？
- 如何避免公开仓库泄露真实业务数据？
- Prisma + SQLite 在本地交付中的优缺点是什么？
- 如果做成正式商业项目，权限、审计、对象存储和部署流水线怎么补？
