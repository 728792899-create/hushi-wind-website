# 胡氏管乐网站完整源码本地运行说明

> 本说明适用于“完整本地运行包”（含 `node_modules` 与本地运行数据库）。公开代码审查包保留虚构品牌素材，但不包含依赖、运行数据库和临时上传文件，需按 README 执行 `npm run install:all && npm run seed:demo` 后再运行。

## 1. 项目内容

本压缩包包含胡氏管乐/上空官网完整本地源码与运行数据：

- `aural-website`：前台官网，Nuxt 4 + Vue 3 + Tailwind CSS。
- `aural-admin`：后台管理端，Vue 3 + Vite + Element Plus。
- `aural-api`：API 后端，Node.js + Express + Prisma + SQLite。
- `aural-api/prisma/dev.db`：本地 SQLite 数据库，包含演示数据和后台账号数据。
- `aural-api/uploads`：本地上传资源和网站图片素材。

已随包保留三个项目的 `node_modules`，另一台电脑通常不需要重新 `npm install`，只要安装 Node.js 即可。

## 2. 运行环境

建议环境：

- Windows 10/11
- Node.js 18 或更高版本（当前打包机器为 Node.js v24.15.0）
- npm 9 或更高版本

检查命令：

```bash
node -v
npm -v
```

## 3. 一键启动方式

解压后进入文件夹，双击：

```text
一键启动-本地开发.bat
```

脚本会打开 3 个命令行窗口，分别启动：

- API 后端：`http://127.0.0.1:1337`
- 前台官网：`http://127.0.0.1:3000`
- 后台管理：`http://127.0.0.1:5175`

等待 20～60 秒后访问：

- 前台官网：`http://127.0.0.1:3000`
- 后台管理：`http://127.0.0.1:5175`
- API 健康检查：`http://127.0.0.1:1337/health`

后台测试账号：`demo_admin`
后台测试密码：`DemoPass_2026!`

## 4. 分别手动启动

如果一键脚本不可用，可以打开 3 个命令行窗口分别执行。

### API 后端

```bat
cd aural-api
set NODE_ENV=development
set DATABASE_URL=file:./dev.db
set PORT=1337
set ADMIN_USERNAME=demo_admin
set ADMIN_PASSWORD=DemoPass_2026!
set ADMIN_TOKEN_SECRET=hushi-demo-local-token-secret-2026-change-me
npm start
```

### 前台官网

```bat
cd aural-website
set NUXT_PUBLIC_API_BASE=http://127.0.0.1:1337
set NUXT_API_INTERNAL_BASE=http://127.0.0.1:1337
set NUXT_PUBLIC_SITE_URL=http://127.0.0.1:3000
set NUXT_PUBLIC_BRAND_IMAGE_BASE=http://127.0.0.1:1337/uploads/real-assets
npm run dev -- --host 127.0.0.1 --port 3000
```

### 后台管理

```bat
cd aural-admin
set VITE_API_BASE=http://127.0.0.1:1337
set VITE_SITE_URL=http://127.0.0.1:3000
npm run dev -- --host 127.0.0.1 --port 5175
```

## 5. 如果依赖损坏或换电脑后启动失败

如果提示找不到模块，可在对应目录重新安装依赖：

```bash
npm install --include=dev
```

## 6. 验证方式

三端启动后，可以双击：

```text
检查本地服务.bat
```

也可以手动访问：

- `http://127.0.0.1:1337/health`
- `http://127.0.0.1:3000`
- `http://127.0.0.1:5175`

## 7. 生产部署提醒

本包用于本地运行、毕业设计展示和源码交接。生产服务器部署时不要继续使用默认弱口令，应更换强密码、配置正式域名、HTTPS、CORS 白名单和 32 位以上的 `ADMIN_TOKEN_SECRET`。
